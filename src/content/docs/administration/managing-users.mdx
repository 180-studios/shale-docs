Shale gives you fine-grained control over who can view, annotate, and approve content in a review session.

## User roles

| Role | Permissions |
|------|-------------|
| **Owner** | Full admin access, billing, org settings |
| **Admin** | Manage users, create sessions, approve |
| **Reviewer** | View sessions, add annotations and comments |
| **Guest** | View and comment via timed link only |

## Inviting users

Go to your organization settings and invite team members by email. They'll receive an invitation to create a Shale account linked to your organization.

## Guest links

For external stakeholders who don't need a full account, generate a **timed guest link** from within any session. Links can be set to expire after a defined period.

## Approver hierarchy

Shale supports structured approval chains — define which roles must approve before a session is considered signed off. All approvals are logged for chain of custody tracking.

## User logging

All session activity (views, annotations, approvals) is logged per user for audit and accountability purposes.
